# COMP5355_Project

Secure Python messenger with TLS transport, CA-issued server certificates, and end-to-end encrypted client messaging.

## Server setup

1. Open a terminal in the project root.
2. Start the server with:

	`python server.py`

3. The first run will automatically create the TLS material under `tls/`:
	- `tls/ca_key.pem`
	- `tls/ca_cert.pem`
	- `tls/server_key.pem`
	- `tls/server_cert.pem`

4. Keep the server running while clients are connecting.

## Client setup

1. Open a second terminal in the project root.
2. Start the client with:

	`python client.py`

3. The client verifies the server using the CA certificate generated by the server.
4. When you register a new account, the client creates a local profile file named:
	- `<username>_profile.json`

	This profile stores the encrypted private key, local salt, trusted public key fingerprints, and replay-tracking data.

## Client menu guide

After launching the client, you will see these options:

### 1. Register Account
- Creates a new user account on the server.
- Generates a new RSA identity key pair for the user.
- Encrypts and saves the private key locally in the profile file.
- Use this first for a new username.

### 2. Login
- Authenticates the user with the server.
- Unlocks the local encrypted private key using the user password.
- Required before sending or receiving messages.

### 3. Logout
- Ends the current client session.
- Clears the loaded private key from memory.

### 4. Send Secure Message
- Sends an end-to-end encrypted message to another registered user.
- The client:
  - fetches the recipient public key from the server,
  - encrypts the message with AES-GCM,
  - protects the AES key with RSA-OAEP,
  - signs the message with RSA-PSS.
- The recipient can verify confidentiality, integrity, and authenticity.

### 5. Check Inbox (Receive)
- Fetches all stored messages for the logged-in user.
- The client verifies the sender signature.
- The client decrypts the AES key and message body.
- Replay detection is applied using message IDs and local history.

### 6. Exit
- Closes the client program.

## Recommended usage flow

1. Start `server.py`.
2. Start `client.py` in one or more terminals.
3. Register a user account in each client profile you need.
4. Log in to the desired account.
5. Send secure messages to other registered users.
6. Use inbox check to receive and verify messages.

## Notes

- The server listens on `127.0.0.1:5555`.
- All client/server traffic is protected by TLS.
- Client-to-client message confidentiality and authenticity are enforced end to end.
